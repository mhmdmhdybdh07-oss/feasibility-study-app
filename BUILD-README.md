# 📦 بناء ملف `.exe` على Windows

<div dir="rtl">

## 🚀 طريقتان للبناء

### 1️⃣ الطريقة الأسهل — ملف Batch (`.bat`)

انقر بزر الفأرة الأيمن على `build.bat` واختر **"Run as administrator"**

أو من موجه الأوامر:
```cmd
build.bat
```

### 2️⃣ طريقة PowerShell (مخرجات أجمل)

انقر بزر الفأرة الأيمن على `build.ps1` واختر **"Run with PowerShell"**

أو من PowerShell:
```powershell
.\build.ps1
```

> 💡 إذا فشل التشغيل بسبب سياسة التنفيذ، شغّل أولاً:
> ```powershell
> Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
> ```

**الوقت الإجمالي المتوقع: 15-25 دقيقة** (حسب سرعة الجهاز والإنترنت)

## ⚠️ استكشاف الأخطاء

### المشكلة: `'node' is not recognized`

**السبب**: Node.js غير مثبت أو ليس في PATH.

**الحل**:
1. ثبّت Node.js من https://nodejs.org (اختر LTS)
2. أعد تشغيل موجه الأوامر
3. تأكد: `node --version`

---

### المشكلة: `npm install` فشل

**الأسباب المحتملة**:
- اتصال إنترنت ضعيف
- ملف `package-lock.json` تالف

**الحلول**:
```cmd
:: حذف lock file وإعادة المحاولة
del package-lock.json
rmdir /s /q node_modules
npm install --legacy-peer-deps
```

---

### المشكلة: `prisma db push` فشل

**السبب**: قد لا يكون مجلد `db` موجوداً.

**الحل**:
```cmd
mkdir db
npx prisma db push
```

---

### المشكلة: `next build` فشل

**الأسباب الشائعة**:
- نقص في الذاكرة: أغلق التطبيقات الأخرى
- خطأ في الكود: راجع رسالة الخطأ

**الحل**:
```cmd
:: تنظيف وإعادة البناء
rmdir /s /q .next
npm run build
```

---

### المشكلة: `electron-builder` فشل

**الأسباب المحتملة**:
- مساحة غير كافية (يحتاج 2 GB)
- مضاد فيروسات يحجب التنزيل
- جدار حماية

**الحلول**:
1. شغّل كمسؤول: انقر يميناً → **Run as administrator**
2. عطّل مضاد الفيروسات مؤقتاً
3. تأكد من مساحة 2 GB فارغة
4. أعد المحاولة (التنزيل قد يفشل أحياناً)

---

### المشكلة: `Could not find any Visual Studio installation`

**السبب**: بعض المكوّنات الأصلية (مثل `@parcel/watcher`) تحتاج Visual Studio Build Tools لتجميعها.

**3 حلول متاحة:**

#### ✅ الحل 1 (افتراضي): تخطي إعادة البناء
الـ `build.bat` المُحدّث يقوم بهذا تلقائياً:
- يضيف `--config.npmRebuild=false`
- يحذف `@parcel/watcher` (مطلوب فقط في وضع التطوير)
- يستخدم البينريات الجاهزة (prebuilt)

إذا استمر الخطأ، جرّب الحل 2.

#### 🔧 الحل 2: تثبيت VS Build Tools
شغّل `install-vs-buildtools.bat` (سكريبت تلقائي) أو:
1. حمّل من: https://visualstudio.microsoft.com/visual-cpp-build-tools/
2. اختر **"Desktop development with C++"**
3. أعد تشغيل الكمبيوتر
4. شغّل `build.bat` مرة أخرى

#### ☁️ الحل 3: استخدم GitHub Actions (موصى به)
- ارفع المشروع إلى GitHub
- شغّل workflow **"Build Desktop App"**
- حمّل `.exe` من **Artifacts**
- **لا حاجة لتثبيت أي شيء محلياً!**

#### 🌐 الحل 4: استخدم PWA بدلاً من .exe
```cmd
npm run dev
```
افتح `http://localhost:3000` في Chrome/Edge → اضغط زر **"تثبيت"** في شريط العنوان.

---

### المشكلة: الملف `.exe` لا يعمل بعد التثبيت

**السبب**: قد يكون مضاد الفيروسات يحجبه.

**الحل**:
1. أضف استثناء في مضاد الفيروسات
2. شغّل كمسؤول
3. تحقق من Windows Defender

---

## 🔄 إعادة البناء بعد التعديلات

بعد تعديل الكود، أعد تشغيل `build.bat` لإنتاج `.exe` جديد.

للبناء السريع (تخطي تثبيت الحزم الموجودة):
```cmd
build.bat
```
(السكريبت يكتشف تلقائياً أن node_modules موجود ويتخطى التثبيت)

---

## 📞 الدعم

إذا فشل البناء، انسخ رسالة الخطأ كاملة وراجع:
1. سجل الأخطاء في ملف `build-error.log` (إن وُجد)
2. تأكد أنك تشغّل السكريبت من مجلد المشروع الصحيح
3. راجع المتطلبات أعلاه
